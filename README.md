### Engineering Philosophy

Good software isn't just code — it solves problems predictably, scales gracefully, and remains understandable long after it's written.

- Build simple systems first. Complexity is a debt you pay back with interest.
- Prefer maintainability over cleverness. Code is read more often than it is written.
- Automate repetitive work. If you do it twice, script it.
- Write software that is observable, testable, and scalable.
